## 2. 下次建议直接做什么

### 第一优先级：做真实桌面环境手工验收

下次最建议直接做这个，不要先继续写代码。

重点验这几个点：

1. 从非条目区域开始拖框是否稳定进入矩形框选
2. 拖框进入列表顶部热区时是否自动向上滚动
3. 拖框进入列表底部热区时是否自动向下滚动
4. 自动滚动过程中，命中结果是否持续更新
5. 到顶部/底部边界时是否停止继续滚动
6. 松开鼠标后是否立即停止自动滚动
7. `Ctrl` 拖框时是否仍保持正确的基线选择语义
8. 空白点击、普通点击、双击、右键菜单有没有被拖框逻辑误伤

如果这一步发现问题，再开下一轮修交互 bug。

### 第二优先级：继续拆 `src/browser.rs`

如果手工验收没有新 bug，下一步最合理的是继续收口 `src/browser.rs` 里还没拆掉的职责。

优先考虑：

- 导航相关状态
- 刷新/目录读取编排
- 文件操作入口
- 剩余 UI 同步逻辑

原则：

- 不要大重构
- 继续按 selection / drag-selection 这轮的方式小步收口
- 保持 `main.rs` 继续只做 wiring
- 保持 `Slint` 只负责输入和显示

### 第三优先级：打磨选择模型

如果你下一轮想继续做交互，而不是做结构整理，建议做：

- 更细的 `Ctrl+Shift` 行为
- 拖框和键盘扩选之间的一致性边界
- 更多贴近桌面文件管理器的选择语义

## 3. 手工验收清单

下次可以直接按这个清单走。

### A. 启动前

- 确认当前在项目根目录：`/app/Files Rusted`
- 确认代码在 `main`
- 先跑一遍：
  - `cargo test`
  - `cargo build`

### B. 启动应用

如果是容器里做启动级验证：

```bash
xvfb-run -a ./target/debug/files-rusted
```

如果是在真实桌面环境：

```bash
./target/debug/files-rusted
```

### C. 拖框自动滚动重点验收

#### 1. 基础拖框
- 从非条目区域按下左键并拖动
- 确认出现拖框 overlay
- 确认命中项按矩形变化更新

#### 2. 顶部自动滚动
- 把拖框持续拉到列表顶部热区
- 确认列表会自动向上滚
- 确认到顶部后停止

#### 3. 底部自动滚动
- 把拖框持续拉到列表底部热区
- 确认列表会自动向下滚
- 确认到内容底部后停止

#### 4. 滚动过程中的命中更新
- 在自动滚动过程中观察选中项是否持续变化
- 确认不是只在鼠标再次移动时才刷新

#### 5. 结束条件
- 松开鼠标
- 确认自动滚动停止
- 确认最终选择结果保留

#### 6. Ctrl 拖框
- 先预选几项
- 按住 `Ctrl` 再拖框
- 确认命中项与基线集合的合并/切换语义正确

#### 7. 非拖框交互回归
- 单击条目
- 双击条目
- 空白点击
- 右键菜单
- 键盘选择
- 确认这些没有被拖框自动滚动破坏

### D. 如果发现问题，优先记录

建议至少记这 4 项：

- 复现步骤
- 实际结果
- 预期结果
- 是否稳定复现
